Лекция 2. Инструкции пересылки. Режимы адресации данных.


Машинная команда — это число, которое кодирует действие, выполняемое процессором.
Операнд — это закодированное описание расположения данного
Режим адресации — способ нахождения данного, закодированного в операнде.

Инструкции пересылки.
Инструкции пересылки являются основными (>70% программ состоит из них). Производят перемещение значение из операнда SRC в операнд DST.

1. Пересылка значения
	mov src, dst	; dst \ [dst]
	
	mov ax, 2	    ; перемещение значения 2 в регистр ax
	mov ax, bx   	; bx → ax 
	mov ax, [bx]    ; разыменование значения по адресу в bx и перемещение значения в ax (в качестве регистра для разыменования можно использовать только регистры bx \ bp и di \ si и их комбинации. 

2. Расчет эффективного адреса.
	lea src, [dst]
	lea ax, [bx+2]     ; bx+2 → ax
	lea ax, [bx+si]    ; bx+si → ax
	lea cx, [bx+si+2]  ; bx+si+2 → cx
	
	
Режимы адресации данных:

    1. Неявный РА — на данные указывает код операции. (По факту безоперандная инструкция). 
       Прим.: CLC
    2. Непосредственный РА — данное занимает место одного из операндов.
       MOV AX, 2
    3. Прямой (абсолютный) РА — эффективный адрес данного занимает место одного из операндов. 
       MOV AX, [2]; MOV AX, [VAR]
    4. Регистровый — данное располагается в командном регистре. 
       MOV AX, BX
    5. Косвенный регистровый РА — адрес данного располагается в определяемом командой регистре. 
       MOV AX, [BX]
    6. Регистровый относительный РА — эффективный адрес данного равен сумме содержимого указанного регистра и смещения хранимого в коде команды.
       MOV AX, [BX + 2] ; DISP [8] \ [16]
    7. Базовый индексный РА — ЕА данного равен сумме содержимых одного из БАЗОВЫХ и одного из ИНДЕКСНЫХ регистров.
       MOV AX, [BX+SI] ; BX \ BP + SI \ DI
    8. Относительный базовый индексный РА — ЕА данного равен сумме содержимых одного из БАЗОВЫХ, одного из ИНДЕКСНЫХ регистров и смещения.
       MOV AX, [BX+SI+4] ; BX \ BP + SI \ DI ; DISP [8] \ [16]
