0. Доработать пример loop.asm, следующим образом: 
	1. Программа должна в бесконечном цикле запрашивать у пользователя проверяемый символ.
	2. Выводить каждые 5 запросов исходную строку.
	3. Завершать программу, если два раза ввели пустой символ (просто нажали ENTER)
	4. Вывод проверок должен быть интерфейсно приятный (напимер: "T - Not found.", где T был введеный символ)


1. Написать программу считывания строки с клавиатуры и а) вывода этой строки в обратном порядке, б) вывода этой строки n раз подряд в цикле.
Все циклы реализовать с помощью инструкций переходов.


2. Написать программу для проверки каждого символа введенной строки (кроме завершающего) на принадлежность к определенному диапазону в таблице ACSII. Диапазон символов запрашивать у пользователя на старте программы.
При первом несовпадении выводить сообщение в формате:
"Some character in the string is not within the specified range." И завершать программу с кодом -1.
Или, если все совпало:
"All characters in a string within the specified range." И завершать программу с кодом 0.

2*. Модифицировать задание 1 таким образом, что программа не останавливается на первом несовпадении, а просматривает всю строку на наличие несовпадений и выводит индексы всех несовпадающих символов и сами несовпадающие символы, при этом собирать информацию об этих несовпадениях в память и выводить эту информацию один раз в конце программы в формате:
"Some character in the string is not within the specified range: [n - S], ... [n - S].", где n - индекс символа в строке, начиная с 1, а S - сам символ.


3. Написать прогрaмму, осуществляющую вывод матрицы (10х10) чисел от 0 до 99 с выравниванием столбцов а) по левому, б) по правому краю.
Пример:
0  1
10 11
20 21

 0  1
10 11
20 21


4. Написать программу, выводящую таблицу ASCII (все 256 символов) в формате:
"символ: hex-значение" (A: 41) в формате XX (два шестнадцатиричных символа).
Таблицу выводить по 8 символов в строке, т.е. всего строк будет 256/8.
Hex-значение получать с использованием строки шестнадцатиричных символов "0123456789ABCDEF".

Примечание: спец. символы (с кодом 09h, 0Ah, 0Dh и тд.) при выводе следует заменить на их "символьное представление" (\t, \n, \r ...) либо на их название (как указано в таблице ASCII), иначе они будут исполнять "свою прямую обязанность" (перенесут строку и тд).

https://www.industrialnets.ru/files/misc/ascii.pdf

Циклы реализовать с помощью инструкций условных \ безусловных переходов.


5. Написать программу, которая делает перевод 2-х байтного числа, лежащего в памяти, в строку (в шестнадцатиричной системе счисления).
Можно использовать следующий алгоритм:
	Итерироваться по каждой "цифре" числа, т.е. по 4 бита. (4 бита получать по маске 1111b, итерироваться по цифрам в числе с помощью сдвигов и операций loop)
	Делать проверку на диапазон (если число < 10, то прибавляем '0', если >= 10, прибавляем 'A')
	Выводить получившийся символ.
